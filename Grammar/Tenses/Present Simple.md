# Present Simple

## Структура / Formula

```
+ I / You / We / They   work
+ He / She / It         works      (добавляем -s / -es)

- I / You / We / They   do not (don't) work
- He / She / It         does not (doesn't) work

? Do    I / you / we / they   work?
? Does  he / she / it         work?
```

### Исключения: he/she/it

| Окончание глагола | Правило | Пример |
|---|---|---|
| -o, -s, -sh, -ch, -x | + **-es** | go → goes, fix → fixes |
| согласная + -y | -y → **-ies** | study → studies |
| остальные | + **-s** | work → works |

Глаголы-исключения: **have → has**, **be → is**

## Когда использовать / When to use

1. **Факты и постоянные состояния** — всегда правда
2. **Привычки и регулярные действия** (часто с: always, usually, often, sometimes, never, every day)
3. **Инструкции и документация** — технические тексты
4. **Расписания** (поезд отправляется, встреча начинается)

## Примеры / Examples

| Русский | English |
|---------|---------|
| Я работаю бэкенд-разработчиком. | I work as a backend developer. |
| Сервер отвечает за 200 мс. | The server responds in 200ms. |
| Функция принимает два аргумента. | The function takes two arguments. |
| Он не понимает проблему. | He doesn't understand the issue. |
| Ты используешь Docker? | Do you use Docker? |

## IT-контекст / In IT work

> Документация, комментарии к коду и описания API почти всегда написаны в Present Simple.

- *The method **returns** a boolean value.*
- *This function **handles** the authentication.*
- *The script **reads** the config file on startup.*
- *I usually **push** to a feature branch first.*
- *We **deploy** every Friday afternoon.*

## Частые ошибки / Common mistakes

- ❌ *He work here* → ✅ *He **works** here*
- ❌ *Do he works?* → ✅ *Does he work?*
- ❌ *I am work* — это другое время ([[Present Continuous]])
- ❌ *He don't know* → ✅ *He **doesn't** know*
