# Modal Verbs

Модальные глаголы не описывают действие — они выражают **отношение** к нему: возможность, разрешение, обязанность, совет.

## Главное правило

```
Subject + modal verb + base verb (инфинитив без to)
```

- ✅ *You should **restart** the server.*
- ❌ *You should to restart* / *You should restarting*

Модальные глаголы **не изменяются**: нет -s для he/she/it, нет -ed, нет -ing.

---

## can / could

### can — умею / могу (в настоящем)

```
I can   help.
Can you review my PR?
I can't reproduce the bug.
```

| Значение | Пример |
|---|---|
| Умение | *I can write Go.* |
| Возможность | *Can we deploy today?* |
| Разрешение (неформально) | *You can use my laptop.* |

### could — прошлое / вежливое / возможность

```
Could you take a look?      ← вежливая просьба (вместо can)
It could be a race condition.  ← возможно (не уверен)
I could fix it, but it'll take time.  ← условная возможность
```

| Значение | Пример |
|---|---|
| Вежливая просьба | *Could you review this PR?* |
| Неуверенная возможность | *It could be a memory leak.* |
| Умение в прошлом | *I could read binary as a kid.* |

---

## should / shouldn't

Совет, рекомендация, правильное с точки зрения говорящего.

```
You should    add error handling.
We shouldn't  deploy on Friday.
Should we     use Redis here?
```

| Значение | Пример |
|---|---|
| Совет | *You should write tests first.* |
| Рекомендация | *We should refactor this.* |
| Вопрос о правильном действии | *Should I open a new ticket?* |

> Не означает обязанность — это мягче. Для строгой обязанности — **must** или **have to**.

---

## must / have to

Строгая обязанность или необходимость.

```
You must   fix this before release.    ← внутренняя уверенность / правило
You have to  get approval first.       ← внешнее требование / процесс
```

| | **must** | **have to** |
|---|---|---|
| Источник | Говорящий сам считает нужным | Внешнее правило / требование |
| Пример | *I must finish this today.* | *We have to follow the style guide.* |

**must not (mustn't)** — строгий запрет:
- *You mustn't push directly to main.*

**don't have to** — необязательно (не запрет!):
- *You don't have to fix it today.* (можно не делать, никто не заставляет)

---

## will / would

**will** — уверенное будущее и обещания (см. [[Future will and going to]]).

**would** — вежливые просьбы, гипотетические ситуации, прошлые привычки.

```
Would you mind reviewing this?    ← очень вежливая просьба
I would use Postgres here.        ← рекомендация / гипотеза
It would be faster with a cache.  ← если бы мы добавили кэш...
```

---

## might / may

Возможность — менее уверенно, чем **can/could**.

```
It might be a config issue.     ← возможно (50% и меньше)
We may release it next week.    ← возможно (чуть официальнее)
```

| | **might** | **could** |
|---|---|---|
| Уверенность | Меньше (~30–50%) | Немного больше |
| Пример | *It might work.* | *It could work.* |

---

## Быстрая таблица / Quick reference

| Глагол | Значение | IT-пример |
|---|---|---|
| can | умею / могу | *I can deploy this.* |
| could | вежливо / возможно | *Could you check the logs?* |
| should | советую / рекомендую | *You should add a retry.* |
| must | обязан (своё мнение) | *We must fix this before prod.* |
| have to | обязан (внешнее) | *We have to pass the audit.* |
| mustn't | строго нельзя | *You mustn't commit secrets.* |
| don't have to | необязательно | *You don't have to do it today.* |
| will | уверенно / обещание | *I'll handle it.* |
| would | вежливо / гипотеза | *I would split this into two PRs.* |
| might/may | возможно | *It might take longer than expected.* |

## Частые ошибки / Common mistakes

- ❌ *You should to check* → ✅ *You should check*
- ❌ *He cans fix it* → ✅ *He can fix it* (нет -s!)
- ❌ *I must to go* → ✅ *I must go*
- ❌ *You don't have to push to main* (звучит как совет, а не запрет) → ✅ *You mustn't push to main*
- ❌ *Could you to send?* → ✅ *Could you send?*
