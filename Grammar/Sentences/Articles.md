# Articles: a / an / the / —

Артикли — одна из главных сложностей для русскоязычных, потому что в русском их нет. Логика простая, но требует практики.

## Алгоритм выбора артикля

```
Есть уточнение после слова?
(who/that/you/we + глагол, of + noun, in this/our...)
        │ да                    │ нет
        ▼                       ▼
       THE            Оба знают о чём речь?
                        │ да        │ нет
                        ▼          ▼
                       THE    Можно посчитать?
                                │ да     │ нет
                                ▼        ▼
                               A/AN       —
```

Используй этот алгоритм когда не уверен — он даёт правильный ответ в большинстве случаев.

---

## a / an — неопределённый артикль

**Когда:** впервые упоминаем что-то, один из многих, не конкретный.

- **a** — перед согласным звуком: *a bug, a server, a function*
- **an** — перед гласным звуком: *an error, an API, an issue, an hour*

> Важно: звук, а не буква. *a **u**nit* (звук [ю]), *an **h**our* (h не произносится).

| Когда | Пример |
|---|---|
| Первое упоминание | *I found **a** bug.* |
| Один из многих | *She is **a** developer.* (одна из многих) |
| Профессия / роль | *He's **a** team lead.* |

---

## the — определённый артикль

**Когда:** оба знают, о чём речь — конкретный объект, уже упомянутый, единственный в своём роде.

| Когда | Пример |
|---|---|
| Уже упомянули раньше | *I found a bug. **The** bug is in the auth module.* |
| Единственный в контексте | ***The** server is down.* (наш сервер) |
| Оба знают что имеется в виду | *Can you check **the** logs?* |
| Уникальный объект | ***The** sun, **the** internet, **the** cloud* |
| Порядковые числительные | ***The** first commit, **the** second PR* |

### Страны и организации с the

Некоторые названия всегда с *the* — как правило, аббревиатуры или слова "соединённый / королевство":

| С the | Без the |
|---|---|
| **the** UK | France |
| **the** USA | Russia |
| **the** Netherlands | Germany |
| **the** Philippines | Japan |
| **the** EU | Canada |

---

## — (нулевой артикль)

**Когда:** говорим о вещи вообще, а не конкретной.

| Когда | Пример |
|---|---|
| Приёмы пищи | *after **lunch**, before **dinner*** |
| Языки | *I speak **Russian** and **English*** |
| Предметы вообще (мн.ч.) | ***Bugs** are inevitable.* (баги вообще) |
| Неисчисляемое вообще | ***Code** should be readable.* |
| Имена людей | ***John** fixed it.* |
| Большинство городов и стран | *in **London**, in **Germany*** |
| by + транспорт | *by **car**, by **plane**, by **train*** |

---

## Главная пара: a → the

Первое упоминание — *a*. Второе и далее — *the*:

> *I found **a** bug yesterday. **The** bug was in the payment module. **The** fix took 2 hours.*

**the при первом упоминании** — тоже возможен, если оба собеседника уже знают о чём речь из контекста (не из текста):

> *We need to discuss **the** deadline.* — оба знают какой дедлайн, даже если первый раз в разговоре

---

## Неисчисляемые — никогда не берут *a*

Эти слова нельзя посчитать → нет *a*, только *—* или *the* (если конкретное):

| Слово             | ❌                 | ✅                               |
| ----------------- | ----------------- | ------------------------------- |
| knowledge         | *a knowledge*     | *knowledge / the knowledge*     |
| information       | *an information*  | *information / the information* |
| advice            | *an advice*       | *advice / some advice*          |
| feedback          | *a feedback*      | *feedback / the feedback*       |
| progress          | *a progress*      | *progress / good progress*      |
| evidence          | *an evidence*     | *evidence / the evidence*       |
| performance       | *a performance\** | *performance / the performance* |
| time (как ресурс) | *a time*          | *time / enough time*            |
| news              | *a news*          | *news / the news*               |
| access            | *an access*       | *access / the access*           |
| approval          | *an approval*     | *approval / the approval*       |
| permission        | *a permission*    | *permission / the permission*   |
| authorization     | *an authorization* | *authorization / the authorization* |

\*исключение: *a performance* = выступление, спектакль (исчисляемое значение)

**Особый случай: time**
- *I don't have **time**.* — время как ресурс → *—*
- *We had **a** great time.* — время как событие/период → *a*
- *This is **the** third time.* — порядковое → *the*

---

## the + уточнение = конкретное

Если после существительного стоит уточнение (*I use / we have / in this ticket*), оно делает его конкретным → *the*:

| Общее (—) | Конкретное (the) |
|---|---|
| *English is hard.* | ***The** English I use at work is technical.* |
| *Code should be clean.* | ***The** code in this function is messy.* |
| *Knowledge is power.* | ***The** knowledge required for this role is specific.* |

---

## "of + noun": the даётся слову ДО of, не после

*"of + noun"* делает конкретным слово **перед** собой. Слово **после** *of* живёт по своим правилам:

```
THE complexity   OF   modern applications
    ↑                 ↑
  берёт the        zero article —
  из-за "of..."    (приложения вообще)
```

| Пример | Почему |
|---|---|
| ***The** complexity of **—** modern apps* | complexity → the (из-за of); apps → — (вообще) |
| ***The** performance of **—** distributed systems* | то же самое |
| ***The** importance of **—** good communication* | communication → — (практика вообще) |
| ***The** quality of **the** code* | code → the (конкретный код) |

Проверяй каждое слово отдельно: слово до *of* и слово после *of* — независимо.

## IT-контекст / In IT work

### Типичные конструкции

| Фраза | Почему |
|---|---|
| *I found **a** bug.* | первое упоминание |
| *Can you fix **the** bug?* | уже знаем о каком баге |
| ***The** server is down.* | наш конкретный сервер |
| *We need **a** new server.* | один из возможных серверов |
| *It might be **a** memory leak.* | одна из возможных причин |
| *I'll check **the** logs.* | конкретные логи нашего сервиса |
| *Open **a** PR when you're done.* | один из будущих PR |
| *Did you review **the** PR?* | конкретный PR, о котором говорили |
| *Write **the** tests first.* | конкретные тесты для этой задачи |
| *Tests are important.* | тесты вообще, как явление — без артикля |

### Профессии и роли

Роль или профессия — всегда *a*:
- *I'm **a** backend developer.*
- *She's **a** QA engineer.*
- *He became **a** team lead last year.*

### Среды — как имена собственные

*production*, *staging*, *development* — названия сред, артикль не нужен:

| ❌ | ✅ |
|---|---|
| *deploy to **the** production* | *deploy to **—** production* |
| *bug in **the** staging* | *bug in **—** staging* |
| *run in **the** development* | *run in **—** development* |

### API, URL, UI

Начинаются на гласный **звук** — нужен *an*:
- ***an** API endpoint*
- ***an** URL* — или ***a** URL* (оба варианта встречаются, URL читают как "you-ar-el" → *a*)
- ***a** UI component* (U читается как [ю] — согласный звук)

## a whole X — целый период

*a whole* + единица времени или работы → всегда **a**:

| ✅ |
|---|
| *I spent **a** whole morning debugging.* |
| *We lost **a** whole week because of that.* |
| ***A** whole sprint was wasted on this.* |

---

## Most / Most of the

| Форма | Когда | Пример |
|---|---|---|
| *Most* + noun | вообще, без конкретизации | *Most bugs are easy to fix.* |
| *Most of the* + noun | конкретная группа | *Most of the bugs we found were critical.* |
| *Most of our/my/their* + noun | наши/мои конкретные | *Most of our deploys went fine.* |
| ❌ *The most of* | — никогда | — |

---

## Притяжательные местоимения заменяют артикль

*my / our / his / her / their* уже указывают на конкретный объект — артикль не нужен:

| ❌ | ✅ |
|---|---|
| *in **the our** architecture* | *in **our** architecture* |
| *reviewed **the my** PR* | *reviewed **my** PR* |
| *fixed **the their** bug* | *fixed **their** bug* |

---

## Быстрая шпаргалка

```
Первый раз упоминаешь?         → a / an
Уже знаем о чём речь?          → the
Говоришь вообще, не конкретно? → —
Приём пищи, язык, имя?         → —
Страна-аббревиатура (UK, USA)?  → the
```

## Единственное число — всегда с артиклем

Исчисляемое существительное в единственном числе **не может стоять без артикля**. Нулевой артикль работает только для множественного числа и неисчисляемых.

| ❌ | ✅ |
|---|---|
| *— team is slow* | ***A** team / **The** team is slow* |
| *— developer fixed it* | ***A** developer / **The** developer fixed it* |
| *— server crashed* | ***A** server / **The** server crashed* |
| *— bug is critical* | ***A** bug / **The** bug is critical* |

Если не знаешь точно — *a* или *the* — задай вопрос: оба знают о чём речь? Да → *the*. Нет → *a*.

---

## Частые ошибки / Common mistakes

- ❌ *I like UK* → ✅ *I like **the** UK*
- ❌ *It's memory leak* → ✅ *It's **a** memory leak*
- ❌ *After **a** lunch* → ✅ *After lunch*
- ❌ *I'm developer* → ✅ *I'm **a** developer*
- ❌ *Can you check logs?* → ✅ *Can you check **the** logs?*
- ❌ *I found **the** bug* (первое упоминание, контекст не установлен) → ✅ *I found **a** bug*
- ❌ *I don't have **a** time* → ✅ *I don't have **—** time*
- ❌ *a knowledge / an advice / a feedback / an approval / an access* → ✅ всегда без *a*
- ❌ *English I use at work* → ✅ ***The** English I use at work* (уточнение делает конкретным)
- ❌ *deploy to **the** production* → ✅ *deploy to **—** production* (среда = имя)
- ❌ *in **the our** system* → ✅ *in **our** system* (притяжательное местоимение уже заменяет артикль)
- ❌ *The most of our issues* → ✅ *Most of our issues* ("the most of" — не существует)
- ❌ *I spent whole day* → ✅ *I spent **a** whole day*
